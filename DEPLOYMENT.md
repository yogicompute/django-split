# DEPLOYMENT.md: Deploying Splitwise App to Production

This guide explains how to deploy the Splitwise app to production on Railway.app (recommended) or Vercel.

---

## 🚀 Recommended: Railway.app (Best for Django)

Railway is perfect for Django apps with databases. Free tier: $5 credit/month.

### Step 1: Create Railway Account & Project

1. Go to https://railway.app
2. Sign up with GitHub
3. Create new project → "Deploy from GitHub repo"
4. Authorize GitHub and select `splitwise_app` repository

### Step 2: Add PostgreSQL Database

1. In Railway dashboard, click "New"
2. Select "PostgreSQL"
3. Railway auto-generates `DATABASE_URL` environment variable ✅

### Step 3: Configure Environment Variables

In Railway dashboard, go to Variables and add:

```env
DEBUG=False
SECRET_KEY=your-random-secret-key-here
ALLOWED_HOSTS=yourapp.railway.app,yourdomain.com
DATABASE_URL=postgresql://...  # Auto-generated by Railway
```

**Generate a new SECRET_KEY:**
```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

### Step 4: Deploy

1. Click "Deploy" in Railway dashboard
2. Choose `Procfile` as start command
3. Wait for deployment (2-3 minutes)
4. Get your live URL from Railway dashboard

### Step 5: Run Initial Migration

In Railway dashboard, go to "Deployments" tab and click the running deployment. Then SSH or use Railway CLI:

```bash
railway run python manage.py migrate
railway run python manage.py createsuperuser
```

Or, if using the start command with auto-migration (already in `railway.json`), migrations run automatically on deploy.

### Step 6: Access Your App

- **App:** `https://yourapp.railway.app`
- **Admin:** `https://yourapp.railway.app/admin` (use superuser credentials)

## 🟦 Alternative: Vercel (Serverless, More Complex)

**⚠️ Warning:** Vercel is not ideal for Django + persistent databases. Use only if you prefer Vercel's ecosystem.

### Challenges with Vercel:
- ❌ Serverless can't persist SQLite
- ❌ Long-running requests timeout
- ❌ CSV imports might fail

### Workaround: Use Vercel + External Database

1. Create a Vercel app
2. Add external PostgreSQL (via Neon, Supabase, etc.)
3. Deploy Django as serverless function

### Alternative Vercel Approach: Deploy on Vercel with Hybrid Stack

If you must use Vercel:

1. **Option A:** Deploy API on Vercel, use serverless functions
2. **Option B:** Use a different hosting (see below)

## 📋 Pre-Deployment Checklist

- [ ] `.gitignore` file created (excludes `.env`, `*.pyc`, `db.sqlite3`)
- [ ] `requirements.txt` includes all dependencies
- [ ] `Procfile` specifies gunicorn start command
- [ ] `runtime.txt` specifies Python version (3.10)
- [ ] `config/settings.py` reads `DEBUG` and `ALLOWED_HOSTS` from environment
- [ ] `SECRET_KEY` is random and unique (not hardcoded)
- [ ] Database URL supports PostgreSQL in production
- [ ] Static files configured with WhiteNoise middleware
- [ ] `manage.py` has execute permissions

---

## 🔐 Security Checklist

Before deploying to production:

- [ ] `DEBUG = False` in production (via environment variable)
- [ ] `SECRET_KEY` is random, unique, and long (50+ characters)
- [ ] `ALLOWED_HOSTS` restricted to your domain(s), not `*`
- [ ] `SECURE_SSL_REDIRECT = True` (forces HTTPS)
- [ ] `SESSION_COOKIE_SECURE = True`
- [ ] `CSRF_COOKIE_SECURE = True`
- [ ] No database credentials in code (use environment variables)
- [ ] Backups configured for database
- [ ] Admin panel (`/admin`) should have strong password

## 📈 Post-Deployment

### 1. Create Superuser

```bash
railway run python manage.py createsuperuser
```

Or via admin panel in your hosting provider's CLI.

### 2. Load Initial Data (Optional)

If you have fixture data:

```bash
railway run python manage.py loaddata fixtures/initial_groups.json
```

### 3. Set Up Backups

- **Railway:** Automatic backups for PostgreSQL ✅
- **Render:** Automatic daily backups ✅
- **PythonAnywhere:** Configure backups in settings

### 4. Monitor Logs

**Railway:**
```bash
railway logs
```

**Render:**
- Dashboard → Logs tab

**PythonAnywhere:**
- Web app → Error logs

### 5. Set Up Custom Domain (Optional)

In your hosting provider's dashboard:
1. Add custom domain (e.g., `splitwise.yourdomain.com`)
2. Update DNS records (usually `CNAME` to hosting provider)
3. Enable SSL certificate (automatic with most providers)

## 🐛 Troubleshooting

### Problem: "No module named 'decouple'"
**Solution:**
```bash
pip install python-decouple
# Then redeploy or run: railway run pip install -r requirements.txt
```

### Problem: "SECRET_KEY is insecure"
**Solution:** Generate a new one:
```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```
Add it to Railway environment variables.

### Problem: Static files not loading (404 on CSS/JS)
**Solution:**
```bash
railway run python manage.py collectstatic --noinput
```

### Problem: Database migrations didn't run
**Solution:** Manual migration:
```bash
railway run python manage.py migrate
```

### Problem: App crashes with "DisallowedHost"
**Solution:** Check `ALLOWED_HOSTS` in Railway environment variables matches your domain.

### Problem: Permission denied on admin panel
**Solution:** Create superuser:
```bash
railway run python manage.py createsuperuser
```

## 📤 Migrating Away from a Platform

To move from one host to another:

```bash
# Export database
./manage.py dumpdata > backup.json

# On new host, restore:
./manage.py migrate
./manage.py loaddata backup.json

# Or for PostgreSQL:
pg_dump old_db > backup.sql
psql new_db < backup.sql
```

## 💾 Backup & Restore

### Backup PostgreSQL Database

**Via Railway CLI:**
```bash
railway run pg_dump $DATABASE_URL > backup.sql
```

**Via Render:**
1. Dashboard → PostgreSQL → Backup
2. Download from backups section

### Restore Database

```bash
railway run psql $DATABASE_URL < backup.sql
```
